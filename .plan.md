## Implementation Steps

### Phase 1: Backup
1. Create backup branches in both repos:
   ```bash
   git -C ~/dev/nixos-config branch backup-before-hyprland-import
   git --git-dir=~/.local/share/dotfiles.git branch backup-before-hyprland-removal
   ```

### Phase 2: Extract Dotfiles Commits as Patches
1. Export all 85 hyprland commits as patches with path rewriting:
   ```bash
   cd ~ && git --git-dir=~/.local/share/dotfiles.git format-patch \
     --stdout -- '.config/hypr' > /tmp/hyprland-patches.mbox
   ```
2. Rewrite paths in patches: `.config/hypr/` → `linux/hyprland/`

### Phase 3: Create Import Branch in nixos-config
1. Create orphan branch starting before hyprland was added
2. Apply the 26 "before" patches (Nov 21-24)
3. Merge with first hyprland commit in nixos-config
4. Apply remaining 59 patches, interleaving by timestamp
5. For the 3 true matches, amend the nixos-config commits to include dotfiles changes

### Phase 4: Rebase onto Master
1. Rebase import branch onto current master
2. Force push (after verification)

### Phase 5: Update Nix Configuration
1. Add `home.file` entries in `linux/hyprland/hyprland.nix`:
   ```nix
   home-manager.users.andrei = {
     home.file.".config/hypr/main.conf".source = ./main.conf;
     home.file.".config/hypr/hyprlock.conf".source = ./hyprlock.conf;
     home.file.".config/hypr/hypridle.conf".source = ./hypridle.conf;
     home.file.".config/hypr/scripts".source = ./scripts;
   };
   ```
2. Remove `source = ./main.conf` from extraConfig

### Phase 6: Clean Dotfiles Repository
1. Use git-filter-repo to remove hyprland:
   ```bash
   git --git-dir=~/.local/share/dotfiles.git filter-repo \
     --invert-paths --path .config/hypr/
   ```

## Risks
- **Force push required**: Both repos need force push after rewrite
- **Conflicts during rebase**: May need manual resolution
- **Broken refs**: Any forks/clones will have diverged history

## Mitigation
- Backup branches created before any destructive operation
- Can restore from backup if anything goes wrong
